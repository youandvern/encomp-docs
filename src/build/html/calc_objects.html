Calculation Objects

class Assumption(assumption)

This is meant to clearly declare important assumptions which form the basis of your calculation template.

Parameters

assumption (str) – The text describing your assumption

>>> a = Assumption("The seismic provisions of ASCE 7-16 control design")
Calculation report will show --> [ASSUME] The seismic provisions of ASCE 7-16 control design
class Calculation(variable_name, expression, unit='', description='', reference='', result_check=False)

This is the primary object used to define a calculation expression or calculated variable.

Parameters

  • variable_name (str) – The symbolic name for the result of this calculation (LaTex formatted)

  • expression (a variable expression [i.e. a+b] or a number) – The calculation expression

  • unit (str, optional) – The physical units of the resulting variable (LaTex formatted), defaults to None

  • description (str, optional) – A text description for the calculation or calculated variable, defaults to None

  • reference (str, optional) – A short text reference (or code reference) to accompany the calculation, defaults to None

  • result_check (bool, optional) – This is used to indicate any Calculation that should be checked as a final result of your calculation template. When set to True, this Calculation will be displayed in the “Results” section of your design portal, defaults to False

>>> a = Input("a",1,'ft')
>>> b = Input('b',4,'ft')
>>> c = Calculation('c', a + b, 'ft')
Calculation report will show --> c = a + b = 1ft + 4ft = 5ft
get_value()

Returns the calculated value of the expression.

Returns

>>> a = Input('a',1,'ft')
>>> b = Input('b',4,'ft')
>>> c = Calculation('c', a + b, 'ft')

class Comparison(a, comparator, b, true_message='OK', false_message='ERROR', description='', reference='', result_check=True)

This is an object used to compare variables or constants as an explicit check in the design process.

Parameters

  • a (Input, Calculation, or a number) – The first variable or number

  • comparator ('<', '<=', '=', '!=', '==', '>', or '>=') – The comparision symbol as a string

  • b (Input, Calculation, or a number) – The second variable or number

  • true_message (str, optional) – The message that should desplay if the Comparison result is true, defaults to “OK”

  • false_message (str, optional) – The message that should desplay if the Comparison result is false, defaults to “ERROR”

  • description (str, optional) – A text description for the comparison, defaults to None

  • reference (str, optional) – A short text reference (or code reference) to accompany the comparison, defaults to None

  • result_check (bool, optional) – This is used to indicate any Comparison that should be checked as a final result of your calculation template. When set to True, this Comparison will be displayed in the “Results” section of your design portal, defaults to False

>>> a = Input("a",1,'ft')
>>> b = Input('b',4,'ft')
>>> Comparison(a, '>', b)
Calculation report will show --> Check a > b
                                    1 ft > 4 ft
                                    --> ERROR
get_value()

Returns the calculated value of the comparison (True or False).

Returns

The result of the evaluated comparison

Return type

bool

>>> a = Input('a',1,'ft')
>>> b = Input('b',4,'ft')
>>> c = Comparison(a, ">", b)
>>> print(c.get_value())
False
class ComparisonForced(a, comparator, b, comparator2=None, c=None, description='', reference='')

This is an object for declaring the result of a comparison. It does not actually compare the values given, but rather displays the comparision exactly as it is given. This can be used to annotate or embellish if/else logic in your calculation templates.

Parameters

  • a (Input, Calculation, or a number) – The first variable or number

  • comparator ('<', '<=', '=', '!=', '==', '>', or '>=') – The comparision symbol as a string

  • b (Input, Calculation, or a number) – The second variable or number

  • comparator2 ('<', '<=', '=', '!=', '==', '>', or '>=', optional) – A second comparision symbol as a string, defaults to None

  • c (Input, Calculation, or a number, optional) – A third variable or number, defaults to None

  • description (str, optional) – A text description for the comparison, defaults to None

  • reference (str, optional) – A short text reference (or code reference) to accompany the comparison, defaults to None

>>> a = Input("a",1,'ft')
>>> b = Input('b',4,'ft')
>>> ComparisonForced(a, ">", b, description="Requirement for passing")
Calculation report will show --> Requirement for passing:
                                    --> a > b
class Heading(text, head_level=1, numbered=True, reference='')

This object can be used to add headings/section titles to calculation reports. If numbered, the heading number will auto-increment. The text size will generally be larger for higher levels (i.e. 1) than lower levels (i.e. 6)

Parameters

  • text (str) – The text for the heading or title

  • head_level (int, optional) – The level of the heading from 1 to 6. Each heading level in its corresponding position would display as 1.2.3.4.5.6. before the heading text, defaults to 1

  • numbered (bool, optional) – Whether the heading should be numbered or not, defaults to True

  • reference (str, optional) – A short text reference (or code reference) to accompany the heading, defaults to None

>>> Heading("Default First Heading")
>>> Heading("Second Level Heading", 2)
>>> Heading("Another Second Level Heading", 2)
>>> Heading("Heading Without Number", head_level=2, numbered=False)
Calculation report will show -->    1.  Default First Heading
                                    1.1.  Second Level Heading
                                    1.2.  Another Second Level Heading
                                    Heading Without Number
class Input(variable_name, default_value=0, unit='', description='', reference='', input_type='number', select_options=None, min_value=None, max_value=None, num_step='any')

This is the primary object used to define an input which can change from calculation to calculation. Inputs may be numbers, text, or select elements with multiple options.

Parameters

  • variable_name (str) – The symbolic name for this input variable (LaTex formatted)

  • default_value (float, int, or str) – The default value for the input which will be overridden when calculation inputs are updated, defaults to 0

  • unit (str, optional) – The physical units of the input variable (LaTex formatted), defaults to None

  • description (str, optional) – A text description for the input variable, defaults to None

  • reference (str, optional) – A short text reference (or code reference) to accompany the input, defaults to None

  • input_type ("number", "select", or "text", optional) – The type of input field used on the design portal; a “select” type must include a list of options to choose from in the select_options parameter, defaults to number

  • select_options (str[], float[], or int[], optional) – A list of options for a select type input variable, defaults to None

  • min_value (float, int, or None, optional) – Set the minimum value a number input is allowed to be, defaults to None

  • max_value (float, int, or None, optional) – Set the maximum value a number input is allowed to be, defaults to None

  • num_step (float, int, or None, optional) – Specifies the interval between legal numbers in a number input field; see https://developer.mozilla.org/en-US/docs/Web/HTML/Attributes/step, defaults to None

>>> a = Input("a",1,'ft')
>>> b = Input("b",4,'ft')
>>> Calculation('c', a + b, 'ft')
Calculation report will show --> c = a + b = 1ft + 4ft = 5ft
get_value()

Returns the value of the input. This will not always return the default value that is set in the template, but will return the updated input value in each unique calculation instance.

Returns

The current value of the input variable

Return type

float, int, or str

>>> a = Input('a',1,'ft')
>>> print(a.get_value())
1
